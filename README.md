# RpaAiClaw 桌面自动化工作台

当前发布版本：`0.1.21`

RpaAiClaw 是基于 Tauri v2、React 19、TypeScript、Rust 和 MCP sidecar 构建的桌面自动化工作台。项目面向企业和个人效率场景，提供可视化流程编排、浏览器自动化、桌面自动化、计划任务、人工交互、AI 辅助和企业资产同步等能力。

## 项目简介

RpaAiClaw 的目标是把零散的自动化工具组织成一个可编排、可运行、可调试、可调度、可追踪的桌面工作台。用户可以把跨系统、重复性、规则明确的操作沉淀为自动化流程，并在流程中组合浏览器操作、桌面操作、本地脚本、MCP 工具、人工确认和 AI 辅助能力。

典型场景包括：

- 网页数据采集、后台系统操作和表格处理。
- 定时报表生成、定时巡检和业务数据同步。
- 批量处理订单、工单、表单、文件和业务记录。
- 将人工确认、验证码输入、异常判断接入半自动流程。
- 团队共享流程、变量、元素和运行记录，沉淀企业自动化资产。

## 技术栈

- 桌面框架：Tauri v2
- 前端框架：React 19、TypeScript、Vite
- 后端能力：Rust、tauri-specta 类型绑定
- 状态管理：组件局部状态、Zustand、TanStack Query
- 自动化扩展：BPMN 流程编排、MCP sidecar、浏览器扩展
- 包管理器：npm

## 打包发布 0.1.21

正式发布前先准备版本号。该命令会同步 `package.json`、`package-lock.json`、`src-tauri/Cargo.toml` 和 `src-tauri/tauri.conf.json`，并跳过自动 Git 提交：

```bash
npm run release:prepare:0.1.21
```

本地打包并生成 `update/latest.json`：

```bash
npm run release:0.1.21:build
```

打包并同步到 Gitee Release 与更新仓库：

```bash
npm run release:0.1.21:full
```

Windows 安装包默认输出到：

```text
src-tauri/target/release/bundle/nsis/
```

发布脚本会自动读取 Tauri 签名私钥。默认路径为：

```text
~/.tauri/rpa-ai-claw.key
```

也可以通过环境变量显式指定：

```bash
TAURI_SIGNING_PRIVATE_KEY=你的签名私钥 npm run release:0.1.21:build
```

RpaAiClaw 是一款面向企业和个人效率场景的桌面自动化工作台。它以流程编排为核心，将浏览器自动化、桌面自动化、AI 辅助、计划任务、人工交互和企业资产同步整合到同一个工作环境中，帮助用户把重复、繁琐、跨系统的操作沉淀为可复用、可调度、可追踪的自动化流程。

软件适用于网页数据采集、后台系统操作、表格处理、文件处理、定时巡检、业务流程辅助、企业内部自动化任务编排等场景。

## 核心定位

RpaAiClaw 的核心能力可以概括为三点：

- 让用户用流程方式设计自动化任务。
- 让流程能够调用浏览器、桌面、移动端、本地脚本和外部工具。
- 让自动化任务具备运行、调试、调度、记录、恢复和企业协同能力。

## 流程编排

软件提供可视化流程编排能力，用户可以将业务步骤设计成自动化流程，并在流程中组合浏览器操作、桌面操作、数据处理、条件判断、变量传递、人工确认和外部工具调用。

流程能力包括：

- 创建、编辑、保存和管理自动化流程。
- 按分类组织流程，便于沉淀常用业务任务。
- 运行流程并查看运行状态。
- 暂停、恢复、停止和调试流程。
- 支持单步调试，方便定位流程异常。
- 记录流程运行历史和执行日志。
- 支持流程运行过程中的人工确认和人工输入。

## 自动化工具集成

RpaAiClaw 内置多类自动化工具能力，流程可以根据业务需要调用不同工具完成具体动作。

主要能力包括：

- 浏览器自动化：网页打开、页面操作、元素定位、输入、点击、截图、脚本执行、网络监控、控制台检查、网页表格提取等。
- 桌面自动化：窗口识别、控件选择、键鼠输入、截图、图像匹配、应用管理、系统状态获取等。
- 文件与数据处理：文件操作、Excel 处理、数据库连接、数据读写、数据转换等。
- 验证码与图像处理：验证码识别、图像匹配、截图分析等。
- 邮件和系统工具：邮件处理、加密解密、本地系统能力调用等。
- 移动端自动化：面向移动应用测试或自动化操作的扩展能力。

这些工具可以被流程统一编排，也可以作为独立能力在工作台中管理和调用。

## 元素资产管理

软件支持将自动化过程中需要反复使用的页面元素、桌面控件和图像元素保存为资产，避免每个流程重复配置。

元素资产可用于：

- 管理网页元素定位信息。
- 管理桌面控件识别信息。
- 管理图像匹配目标。
- 按分类维护不同业务系统的元素。
- 在流程中复用元素，提高流程稳定性和维护效率。

## 变量与数据管理

RpaAiClaw 支持流程变量和运行变量，用户可以在流程执行过程中传递数据、保存中间结果、控制条件分支，并在运行记录中追踪关键数据。

变量能力包括：

- 创建和维护流程变量。
- 按分类管理变量。
- 在流程节点之间传递变量值。
- 保存运行时变量快照。
- 支持运行过程中更新和查看变量。

## 计划任务

软件支持将自动化流程配置为计划任务，让流程按指定时间或规则自动执行。

计划任务能力包括：

- 创建、修改和删除计划任务。
- 手动立即触发任务。
- 查看任务执行记录。
- 跟踪任务成功、失败和运行状态。

适合用于定时报表生成、定时数据采集、定时巡检、定时同步等场景。

## AI 辅助工作区

RpaAiClaw 提供 AI 工作区，用于辅助用户分析问题、生成思路、处理流程上下文和提升自动化设计效率。

AI 能力包括：

- 配置不同 AI 服务提供方。
- 验证 AI 服务连接。
- 维护 AI 对话会话。
- 保存和管理对话消息。
- 辅助流程设计、问题排查和文本处理。

## 人工交互

自动化流程并不总是完全无人值守。RpaAiClaw 支持在流程执行到关键节点时暂停并等待人工处理。

人工交互适用于：

- 需要人工确认审批结果。
- 需要用户输入验证码、账号信息或业务参数。
- 需要人工判断异常情况后继续执行。
- 需要在半自动流程中保留关键决策权。

## 多窗口协作

软件采用多窗口协作方式，不同任务可以在独立窗口中完成，减少主工作台拥挤。

多窗口能力包括：

- 主工作台窗口，用于管理流程、任务、资产和系统能力。
- 快捷面板，用于快速唤起常用操作。
- 人工交互窗口，用于流程中的用户确认和输入。
- 运行态窗口，用于查看流程执行会话。
- AI 参考窗口，用于辅助流程设计和问题分析。
- 浏览器动作录制窗口，用于采集网页操作步骤。
- 桌面动作录制窗口，用于采集桌面操作步骤。
- 表格提取窗口，用于采集和预览网页表格数据。

## 企业协同与云端资产

RpaAiClaw 支持企业场景下的登录、授权、云端流程和云端资产管理，便于团队共享自动化成果。

企业能力包括：

- 企业服务连接。
- 企业登录和会话维护。
- 云端流程查看、获取和发布。
- 云端变量分类和变量管理。
- 云端元素分类和元素管理。
- 企业消息连接状态管理。
- 与外部业务系统同步集成资源。

## 授权与安全

软件包含授权管理能力，支持本地和在线授权场景。

授权能力包括：

- 查看授权状态。
- 在线激活。
- 导出设备硬件信息。
- 检测硬件变化。
- 反激活授权。
- 导出授权备份。
- 查看激活历史。

## 日志、回收站与恢复

RpaAiClaw 提供运行日志、回收站和异常恢复能力，帮助用户追踪问题并降低误操作风险。

相关能力包括：

- 查看运行日志。
- 搜索日志内容。
- 统计日志信息。
- 清理日志文件。
- 恢复被删除的数据。
- 彻底清理回收站数据。
- 保存和读取异常恢复数据。

## 本地服务与外部接入

软件支持本地服务能力，便于外部系统或本地工具与工作台联动。

本地服务能力包括：

- 启动和停止本地服务。
- 管理访问令牌。
- 重置访问令牌。
- 查看服务运行状态。
- 为外部调用提供本地接入入口。

## 更新与系统能力

RpaAiClaw 提供桌面应用常见的系统级能力，保障软件可持续使用和维护。

系统能力包括：

- 应用内更新。
- 安装包下载与启动安装。
- 系统托盘。
- 单实例启动。
- 窗口状态保存。
- 开机自启动。
- 原生系统通知。
- 系统对话框。

## 典型使用场景

- 自动登录业务系统并采集网页数据。
- 批量处理后台表单、订单、工单或报表。
- 定时从网页、文件或数据库中获取数据并生成结果。
- 将多个系统中的重复操作编排成自动化流程。
- 在流程中加入人工确认，形成半自动业务处理链路。
- 将团队常用流程、变量和元素同步到企业侧统一管理。
- 使用 AI 辅助分析流程异常或生成自动化处理思路。

## 产品价值

RpaAiClaw 的价值在于把零散的自动化能力组织成完整工作台：

- 对业务用户来说，可以把重复操作变成可复用流程。
- 对技术用户来说，可以通过工具扩展和流程编排组合复杂能力。
- 对团队来说，可以沉淀流程、变量、元素和执行记录，形成可协作的自动化资产。
- 对企业来说，可以逐步建设内部自动化能力中心，降低重复劳动和流程维护成本。
